[[contract-stubs]]
== Contract Stubs

Stubborn publishes a stubs JAR that consumers can use via Spring Cloud Contract Stub Runner to test against the broker's API without running the actual server.

=== Stubs JAR Contents

The stubs JAR (`stubborn-{version}-stubs.jar`) contains:

* **WireMock JSON mappings** (`META-INF/.../mappings/*.json`) -- for request/response stub matching
* **Groovy Contract DSL** (`META-INF/.../contracts/*.groovy`) -- for contract verification

These are generated automatically from contract tests using Spring REST Docs + `SpringCloudContractRestDocs.dslContract()`.

=== Available Stubs

[cols="1,1,1,1",options="header"]
|===
| Stub Name | Method | Path | Description

| `register-application` | POST | `/api/v1/applications` | Register a new application
| `get-application` | GET | `/api/v1/applications/{name}` | Get application by name
| `list-applications` | GET | `/api/v1/applications` | List all applications (paginated)
| `delete-application` | DELETE | `/api/v1/applications/{name}` | Delete an application
| `publish-contract` | POST | `/api/v1/applications/{name}/versions/{version}/contracts` | Publish a contract
| `get-contract` | GET | `/api/v1/applications/{name}/versions/{version}/contracts/{id}` | Get a contract
| `list-contracts` | GET | `/api/v1/applications/{name}/versions/{version}/contracts` | List contracts
| `record-verification` | POST | `/api/v1/verifications` | Record a verification result
| `list-verifications` | GET | `/api/v1/verifications` | List verification results
| `record-deployment` | POST | `/api/v1/deployments` | Record a deployment
| `get-deployment` | GET | `/api/v1/deployments/{environment}/{application}` | Get a deployment
| `list-deployments` | GET | `/api/v1/deployments/{environment}` | List deployments
| `can-i-deploy-safe` | GET | `/api/v1/can-i-deploy` | Can-I-Deploy check (safe result)
| `can-i-deploy-unsafe` | GET | `/api/v1/can-i-deploy` | Can-I-Deploy check (unsafe result)
|===

=== Using Stubs with Stub Runner

==== Maven Dependency

[source,xml]
----
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>stubborn</artifactId>
    <version>${broker.version}</version>
    <classifier>stubs</classifier>
    <scope>test</scope>
</dependency>
----

==== In Tests

[source,java]
----
@SpringBootTest
@AutoConfigureStubRunner(
    ids = "sh.stubborn:stubborn:+:stubs:8090",
    stubsMode = StubRunnerProperties.StubsMode.LOCAL
)
class BrokerClientTest {

    @Test
    void should_register_application_via_broker_stub() {
        RestClient client = RestClient.builder()
            .baseUrl("http://localhost:8090")
            .build();

        String response = client.post()
            .uri("/api/v1/applications")
            .contentType(MediaType.APPLICATION_JSON)
            .body("""
                {"name": "my-service", "description": "My service", "owner": "team-a"}
                """)
            .retrieve()
            .body(String.class);

        assertThat(response).contains("my-service");
    }
}
----

==== Programmatic Stub Runner

[source,java]
----
StubRunner stubRunner = new StubRunner(
    StubRunnerOptionsBuilder.builder()
        .withStubsFromClasspath("sh.stubborn:stubborn")
        .withPort(8090)
        .build()
);
stubRunner.runStubs();
----

=== Building Stubs JAR

The stubs JAR is built automatically during `./mvnw verify`:

[source,bash]
----
./mvnw clean verify -pl broker
ls broker/target/*-stubs.jar
# broker/target/stubborn-0.1.0-SNAPSHOT-stubs.jar
----

The contract tests in `broker/src/test/java/.../broker/*ContractTest.java` classes generate both WireMock JSON and Groovy DSL via `SpringCloudContractRestDocs.dslContract()`, then `maven-assembly-plugin` in the broker module packages them.
